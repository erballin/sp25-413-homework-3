## Assignment 3: WordPress Child Theme

### Child Themes

A child theme in WordPress is a theme that inherits the functionality and styling of another theme, called the parent theme. Instead of editing the parent theme directly, you make changes in the child theme so your work is not lost when the parent theme updates. This is the recommended method for customization because it keeps your changes safe and separate from the original files. The Template: field in the child theme’s style.css connects it to the parent theme by telling WordPress which theme to load as the base.

### Template Hierarchy

WordPress uses a system called the template hierarchy to decide which file to use when displaying content. It looks for the most specific template first (like single.php for posts), and if it doesn’t find one, it falls back to more general files (like index.php). When using a child theme, WordPress checks the child theme’s files first before looking in the parent theme. This means any template placed in the child theme will override the parent version. For example, I created a custom single.php in my child theme, so WordPress uses my modified layout instead of the parent theme’s default post layout.

### Real-World Reflection

I modified the single.php layout by adding a featured image and moving the author and date to the bottom of the post. This improves the reading experience by making the content the main focus while still showing important information in a clear place. It also makes posts feel more modern and visually engaging for readers of The Metro Report.
